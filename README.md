# ❄️ 온라인 에어컨 (Online Air Conditioner)

이 프로젝트는 웹 브라우저에서 작동하는 가상 에어컨 시뮬레이터입니다.  
별도의 서버 없이 **순수 HTML, CSS, JavaScript**로 제작되어 가볍고 빠르며, **Cloudflare Pages** 또는 임의의 정적 호스팅 서비스에 바로 배포할 수 있습니다.

- **라이브 데모**: Cloudflare Pages에 폴더를 업로드하여 즉시 호스팅 가능합니다.

---

## 주요 기능 (Features)

1. **실감나는 에어컨 연출**:
   - 에어컨 전원 ON/OFF에 따른 송풍구 플랩 작동(열림, 닫힘, 자동 스윙).
   - 에어컨 가동 모드(냉방 ❄️, 난방 ☀️, 제습 💧, 송풍 🌀)에 따른 LED 디스플레이 및 배경 광채 변화.
   - 설정 풍량(자동, 약, 중, 강) 및 특수 모드에 맞게 유기적으로 흩날리는 바람 애니메이션.

2. **Web Audio API 기반 100% 로컬 오디오 합성**:
   - **외부 음원 파일 의존성 없음**: 브라우저의 오디오 합성 기능을 활용하여 오프라인에서도 끊김 없이 작동합니다.
   - 에어컨 작동음에 맞는 저주파 바람 소리 및 LFO(저주파 발진기)를 활용한 입체감 있는 바람 세기 변화 연출.
   - 실제 리모컨 버튼음을 재현한 다양한 비프음 및 무음 모드 지원.
   - 귀여운 이스터 에그: 하단의 자고 있는 고양이를 클릭하면 귀여운 야옹 소리 재생.

3. **실시간 실내 온도 시뮬레이션**:
   - 실외 온도와 에어컨 가동 상태 간의 열역학적 변화(실내외 열교환 및 에어컨 성능)를 매초 시뮬레이션합니다.
   - 에어컨이 꺼지면 실외 온도 영향으로 실내 온도가 점차 상승하고, 켜지면 설정 온도를 향해 쾌적하게 내려갑니다.

4. **누적 전기세 및 전력 계산기**:
   - 선택한 가동 모드, 풍량, 에코/터보 설정에 맞춰 실시간 전력 소비량(W)을 도출합니다.
   - 현대식 인버터 에어컨 방식을 차용하여, 실내 온도가 설정 온도 근처에 도달하면 전력 소비량이 대폭 감소하는 메커니즘을 내장했습니다.
   - 시뮬레이션 배속(1초 = 1분)을 적용하여 실시간으로 쌓이는 누적 전력량(kWh)과 예상 전기세를 한국어(KRW) 및 영어(USD) 환경에 맞추어 보여줍니다.

5. **감성적인 주변 환경 테마 & 한국어/영어 로컬라이징**:
   - 우측 상단 테마 버튼을 통해 낮(☀️), 노을(🌇), 밤(🌙) 모드를 전환할 수 있으며, 테마에 맞추어 실내 방 안의 조명과 창밖 배경이 아름답게 변화합니다.
   - 리모컨 및 화면의 언어를 한국어와 영어로 완전하게 전환할 수 있습니다.

## 실행 및 배포 방법 (How to Run & Deploy)

### 1. 로컬에서 실행하기
본 프로젝트는 외부 빌드 과정이나 의존성 패키지가 전혀 없는 정적 페이지입니다.
- `index.html` 파일을 크롬, 사파리, 엣지 등 현대식 브라우저로 직접 더블 클릭하여 실행할 수 있습니다.
- 또는 VS Code의 `Live Server` 익스텐션을 사용하여 로컬 호스트로 실행하면 더욱 안정적으로 테스트 가능합니다.

> 💡 **참고**: 최신 브라우저 정책상 사용자 상호작용(리모컨 버튼 클릭 등)이 발생하기 전에는 효과음 재생이 제한됩니다. 화면을 한 번 클릭하거나 리모컨 버튼을 누르면 사운드가 정상 작동하기 시작합니다.

### 2. Cloudflare Pages에 배포하기
Cloudflare Pages를 이용하면 1분 안에 무료로 웹상에 배포할 수 있습니다.

1. [Cloudflare 대시보드](https://dash.cloudflare.com/)에 로그인합니다.
2. **Pages** 메뉴로 이동한 뒤 **프로젝트 만들기 (Create a project)**를 클릭합니다.
3. **직접 업로드 (Direct Upload)**를 선택하고 프로젝트 이름을 입력합니다.
4. 이 저장소 폴더 전체를 드래그 앤 드롭하여 업로드합니다.
5. 업로드가 완료되면 제공되는 고유 서브도메인(예: `프로젝트명.pages.dev`)으로 접속하여 온라인 에어컨을 즐기실 수 있습니다.
6. (또는 GitHub 저장소와 연동하여 자동으로 푸시할 때마다 자동 배포되도록 구성할 수도 있습니다. 빌드 설정 단계에서 **Build command** 및 **Output directory**를 비워두시면 됩니다.)
